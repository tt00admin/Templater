# templater 仕様書

## 1. 概要

`templater` は、VS Code 上でプロジェクト用テンプレートを一元管理し、任意のワークスペースへ安全に反映するための拡張機能である。

対象テンプレートは、`AGENT.md`、設定ファイル、プロジェクト標準のディレクトリ構造、各種 Markdown ファイル、スクリプト、設定スニペットなどを想定する。ユーザーはテンプレートをカテゴリ別に管理し、差分を確認しながら現在のプロジェクトへ適用できる。

## 2. 解決したい課題

テンプレートが複数のリポジトリ、ローカルディレクトリ、メモ、過去プロジェクトに散らばると、次の問題が起きる。

- どのテンプレートが最新か分からない。
- プロジェクトごとの初期セットアップで手作業コピーが増える。
- `AGENT.md` や config 系ファイルの更新漏れが起きる。
- テンプレートの差分確認や上書き判断が面倒。
- チーム標準テンプレートと個人テンプレートの使い分けが難しい。

`templater` は、テンプレートの保存場所、検索、編集、適用、差分確認を VS Code 内に集約する。

## 3. 想定ユーザー

- 複数プロジェクトを扱う個人開発者
- AI コーディング支援用の `AGENT.md` や指示ファイルを使い回す開発者
- チーム標準の設定ファイルやディレクトリ構成を管理したい開発チーム
- プロジェクト初期化の手作業を減らしたい開発者

## 4. 用語

- テンプレート: 反映対象となるファイル、ディレクトリ、またはファイル群。
- テンプレートセット: 複数テンプレートをまとめた単位。例: `node-typescript`, `ai-agent-docs`, `python-cli`。
- ソース: テンプレートの保管元。ローカルディレクトリ、拡張機能内ストレージ、Git リポジトリを段階的に想定する。
- 適用: テンプレートを現在のワークスペースへコピーまたは生成する処理。
- 変数: テンプレート内で置換される値。例: `{{projectName}}`, `{{author}}`。

## 5. スコープ

### 5.1 MVP に含める

- ローカルディレクトリをテンプレートソースとして登録する。
- 登録済みテンプレートセットを VS Code の Tree View に表示する。
- テンプレートセットの内容を確認する。
- テンプレートを現在のワークスペースへ適用する。
- 適用前に既存ファイルとの衝突を検出する。
- 衝突時に、スキップ、上書き、別名作成を選べる。
- テンプレート内の基本変数を置換する。
- コマンドパレットから主要操作を実行できる。
- 設定画面でテンプレートソース一覧を管理できる。

### 5.2 MVP では含めない

- リモート Git テンプレートソースの同期
- マーケットプレイス形式のテンプレート配布
- 複雑なテンプレート条件分岐
- GUI でのドラッグアンドドロップ編集
- チーム権限管理

これらは将来拡張として扱う。

## 6. 機能要件

### 6.1 テンプレートソース管理

- ユーザーはローカルディレクトリをテンプレートソースとして追加できる。
- 複数のテンプレートソースを登録できる。
- 登録済みソースは VS Code 設定に保存する。
- 存在しないパス、読み取り不能なパスはエラーとして表示する。

設定例:

```json
{
  "templater.sources": [
    {
      "name": "personal",
      "path": "/Users/me/templates"
    }
  ]
}
```

### 6.2 テンプレートセット構造

テンプレートソースとして追加したフォルダ自体、配下の各フォルダ、配下の各ファイルをテンプレートセットとして扱う。ユーザーが Add でフォルダを選択した場合、そのフォルダ自体をすぐ適用可能なテンプレートとして表示する。

登録したフォルダ自体を反映するために、登録フォルダ配下へ同名フォルダを置く必要はない。登録フォルダ直下にある同名フォルダは旧 workaround とみなし、テンプレート一覧および登録フォルダ本体の適用対象から除外する。

```text
templates/
  ai-agent-docs/
    template.json
    AGENT.md
    .codex/
      config.toml
  node-typescript/
    template.json
    package.json
    tsconfig.json
    src/
      index.ts
```

フォルダテンプレートでは、そのフォルダ直下の `template.json` が任意のメタデータとして使われる。`template.json` 自体は適用対象ファイルとして扱わない。

ファイルテンプレートでは、対象ファイル1つだけを現在のワークスペース直下へ適用する。例: `templates/AGENT.md` は `AGENT.md` テンプレートとして表示され、適用するとワークスペース直下に `AGENT.md` を作成する。

```json
{
  "name": "ai-agent-docs",
  "displayName": "AI Agent Docs",
  "description": "AGENT.md and AI coding assistant configuration files.",
  "variables": [
    {
      "name": "projectName",
      "label": "Project name",
      "default": "{{workspaceName}}",
      "required": true
    }
  ],
  "ignore": [".DS_Store"]
}
```

### 6.3 テンプレート一覧表示

- Activity Bar に `templater` ビューを追加する。
- Tree View でソース、テンプレートセット、ファイルを階層表示する。
- テンプレートセットには説明、ソース名、ファイル数を表示する。
- テンプレートセットを右クリックして適用できる。

### 6.4 テンプレート適用

- 現在開いているワークスペースのルートへ適用する。
- ワークスペースが未選択の場合はエラーを表示する。
- 適用前に作成予定ファイルの一覧を計算する。
- 既存ファイルと衝突する場合はユーザーに処理方針を確認する。
- 適用完了後、作成・更新・スキップされたファイル数を通知する。

衝突時の選択肢:

- Skip: 既存ファイルを保持する。
- Overwrite: テンプレートで上書きする。
- Rename: `filename.template` などの別名で作成する。
- Cancel: 適用処理を中止する。

### 6.5 差分確認

- 衝突ファイルについて、テンプレート内容と既存ファイルの差分を VS Code diff editor で確認できる。
- MVP では差分確認はコマンドとして提供し、適用フロー内では衝突検出と方針選択を優先する。

### 6.6 変数置換

テンプレートファイル内の `{{variableName}}` を適用時に置換する。

組み込み変数:

- `{{workspaceName}}`: 現在のワークスペース名
- `{{date}}`: 適用日の `YYYY-MM-DD`
- `{{year}}`: 適用年

ユーザー定義変数:

- `template.json` の `variables` に定義する。
- 必須変数は適用時に入力を求める。
- default がある場合は入力初期値として使う。

### 6.7 コマンド

提供する VS Code コマンド:

- `templater.addSource`: テンプレートソースを追加する。
- `templater.removeSource`: テンプレートソースを削除する。
- `templater.refresh`: テンプレート一覧を再読み込みする。
- `templater.applyTemplate`: 選択したテンプレートセットを適用する。
- `templater.previewTemplate`: テンプレートセットの内容を表示する。
- `templater.openTemplateSource`: テンプレートソースを開く。

### 6.8 設定

提供する設定:

- `templater.sources`: テンプレートソース一覧。
- `templater.defaultConflictAction`: 既定の衝突処理。`ask`, `skip`, `overwrite`, `rename`。
- `templater.enableVariableReplacement`: 変数置換を有効にするか。
- `templater.renameSuffix`: Rename 時に付与する suffix。既定値は `.template`。

## 7. 非機能要件

- ワークスペース外への書き込みは行わない。
- シンボリックリンクは MVP では辿らない。
- バイナリファイルは変数置換せず、そのままコピーする。
- 大量ファイルでも UI をブロックしないよう、非同期 API を使う。
- 適用処理中にエラーが起きた場合は、失敗ファイルと理由を表示する。
- 上書き操作はユーザーの明示的な選択なしに実行しない。ただし設定で `overwrite` を既定にした場合を除く。

## 8. アーキテクチャ

### 8.1 主要モジュール

- `extension.ts`
  - 拡張機能の起動、コマンド登録、ビュー登録。
- `config/templateSources.ts`
  - VS Code 設定からテンプレートソースを読み書きする。
- `templates/templateScanner.ts`
  - ソース配下を走査し、テンプレートセットを検出する。
- `templates/templateManifest.ts`
  - `template.json` の読み込み、検証、デフォルト値補完。
- `views/templateTreeProvider.ts`
  - Tree View のデータプロバイダー。
- `apply/templateApplier.ts`
  - 適用計画の作成、衝突検出、コピー、変数置換。
- `apply/conflictResolver.ts`
  - 衝突時の方針決定。
- `apply/variableResolver.ts`
  - 組み込み変数とユーザー入力変数の解決。
- `utils/fileSystem.ts`
  - VS Code Workspace FS API を使うファイル操作ユーティリティ。

### 8.2 データフロー

1. 拡張機能起動時に設定からテンプレートソースを読み込む。
2. Tree View がテンプレートソースをスキャンする。
3. ユーザーがテンプレートセットを選択し、適用コマンドを実行する。
4. `templateApplier` が適用計画を作成する。
5. 衝突がある場合、`conflictResolver` が処理方針を取得する。
6. `variableResolver` が変数値を確定する。
7. ファイル作成、ディレクトリ作成、コピー、置換を実行する。
8. 結果を通知し、必要に応じて作成ファイルを開く。

## 9. エラーハンドリング

- テンプレートソースが存在しない: Tree View に警告アイテムを表示する。
- `template.json` が壊れている: 該当テンプレートを警告付きで表示し、適用時に詳細を出す。
- ワークスペース未選択: 適用コマンドを中止し、ワークスペースを開くよう通知する。
- 書き込み権限なし: 対象パスとエラー内容を表示する。
- 変数入力キャンセル: 適用処理を中止する。

## 10. UX 方針

- 主要操作は Tree View の右クリックと Command Palette の両方から実行できる。
- 破壊的操作である上書きは明示的に確認する。
- 通知は短くし、詳細は Output Channel に出す。
- テンプレートの管理作業は通常の VS Code ファイル編集体験に寄せる。
- テンプレートソース追加は Folder Picker を使う。

## 11. テスト方針

- `templateScanner` はユニットテストでテンプレート検出を確認する。
- `templateManifest` は正常系、不正 JSON、任意フィールド欠落を確認する。
- `variableResolver` は組み込み変数、デフォルト値、必須入力を確認する。
- `templateApplier` は適用計画、衝突検出、skip/overwrite/rename を確認する。
- VS Code 拡張の統合テストでコマンド登録と Tree View 表示を確認する。

## 12. 将来拡張

- Git リポジトリをテンプレートソースとして登録し、pull/sync できるようにする。
- テンプレートセットをタグ、カテゴリ、検索で絞り込めるようにする。
- テンプレート適用前のチェックリストを定義できるようにする。
- 条件分岐やファイル名変数置換をサポートする。
- テンプレート適用履歴を保存し、再適用や差分更新を支援する。
- チーム共有用の推奨テンプレートセットをワークスペース設定で配布できるようにする。
